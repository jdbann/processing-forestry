Get Tasks

Now there’s not much point having these lumberjacks standing around. They get demotivated and start wandering around. In fact, they’re so desperate to find something to do that they will constantly ask you for tasks.

GET /people/24f2caed-0fe4-48e6-adfc-a2ea2cd730cd/tasks
Host: localhost
Port: 4567
Accept: application/json

You’ll notice that they each ask for their own tasks by adding their employee id to the path: /people/:person_id/tasks.

Speaking their language

You’ll notice that in this request, they’re specifically saying that they only accept application/json. So they’re looking for a valid JSON object in the body of your response.

More specifically, they’re expecting this returned JSON object to include a tasks key with an array:

{
  "tasks": []
}

How to succeed

For now, we just need the lumberjacks to know that they are getting tasks correctly, but there isn’t anything for them to do.

So, return a valid response when the lumberjack asks for new tasks.
